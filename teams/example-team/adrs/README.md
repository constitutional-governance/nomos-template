# Team ADRs for `example-team`

Architectural decisions scoped to this team. Use this for:
- Decisions that apply only to resources owned by this team
- Team-internal constraints not relevant to the broader domain
- Choices that are too specific for a domain-level ADR

Follow the same format as domain ADRs:

```markdown
# ADR-001: Title

**Status:** Proposed | Accepted | Superseded | Deprecated
**Date:** YYYY-MM-DD

## Decision
## Rationale
## Alternatives rejected
## Consequences
```

Name files `NNN-title.md` (e.g. `001-pos-infix-required.md`).
